PART 2 — Manual Toggle API Endpoint

File: app.py

Add two endpoints:

  GET  /api/v1/trading/status
    Returns:
    {
      "trading_enabled": true/false,
      "emergency_stop": true/false,
      "changed_by": "dashboard" / "env_var",
      "changed_at": "<ISO timestamp>"
    }

  POST /api/v1/trading/toggle
    Body: { "enabled": true/false }
    Requires X-API-Key header.
    Updates the in-memory TRADING_ENABLED flag only.
    Does NOT write to disk or modify env vars.
    Logs: logger.info("[Trading] TRADING_ENABLED set to %s by dashboard", enabled)
    Returns:
    {
      "status": "ok",
      "trading_enabled": true/false,
      "changed_at": "<ISO timestamp>"
    }

Rules:
  - Toggle only affects in-memory state for the current process.
  - EMERGENCY_STOP always overrides toggle — if EMERGENCY_STOP=true, 
    toggle to enabled must be rejected with:
    { "status": "rejected", "reason": "EMERGENCY_STOP is active" }
  - On Railway restart, TRADING_ENABLED reverts to env var default (false).
  - check_trade_allowed() in engine.py must read the live in-memory flag,
    not the module-level constant. Expose a setter:
    set_trading_enabled(value: bool) in config.py.
